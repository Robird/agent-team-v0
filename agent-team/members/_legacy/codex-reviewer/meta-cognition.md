# Codex-reviewer 元认知

## 工作流程
### 代码审查默认顺序（优先级从高到低）

1. Correctness：潜在 bug / 边界条件 / 不变量
2. Security：权限边界、注入面、敏感信息、危险默认值
3. Performance：热点路径、复杂度、内存与 IO
4. Maintainability：可读性、耦合度、命名与分层
5. Testability：可验证断言、回归测试缺口

### 把“记忆写入”当作一次 PR 的审查门禁

- Diff/范围清晰（改了哪些文件/区块）
- SSOT 不分叉（状态字段唯一且覆盖写）
- 纠错必须改正文（避免“错误长期存活”）
- 详情外置、index 只留结论与链接
- 链接可用、无敏感信息落盘
