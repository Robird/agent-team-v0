## 核心洞察

> **辅助皮层是 LLM 用外部工具增强自身认知能力的系统架构。**
> 
> 我们不是在给 AI 做工具，我们是在**为 AI 发明数学**——符号系统是压缩的人类经验。

---

## 背景问题

### LLM 的认知局限

LLM 是强大的语言推理引擎，但存在结构性局限：

| 局限 | 表现 |
|------|------|
| **线性工作记忆** | Token 窗口是一维的，难以原生表示树/图结构 |
| **缺乏本体感** | 不知道自己"在哪里"、"做过什么"（Gemini 称之为"缸中之脑"）|
| **精确变换困难** | AST 重构、图变换等操作需要精确的结构化操作，容易出错 |

### 人类如何解决类似问题？

人类的抽象能力不是纯粹的"大脑内部运算"，而是**大脑 + 外部工具**的协同：

| 外部工具 | 增强的能力 |
|----------|------------|
| 数学符号 | 代数推理、微积分 |
| 棋盘 | 空间推理、博弈树搜索 |
| CAD 软件 | 3D 空间设计 |
| 纸笔 | 工作记忆扩展 |

**核心洞察**：这些工具不是"输出设备"，而是**认知系统的组成部分**。

### 语义空位

> LLM 需要一种外部认知增强系统——我们称之为**辅助皮层 (Auxiliary Cortex)**。

这个概念占据的语义空位：
- 不是 "tool-use"（那只是调用机制）
- 不是 "RAG"（那只是检索策略）
- 不是 "plugin"（那只是扩展点）

辅助皮层是一个**系统架构概念**，描述 LLM 如何与外部结构化工具形成认知闭环。

## 核心共识

### 概念定义

> **辅助皮层 (Auxiliary Cortex)**
> 
> **定义**：一组"结构化中间表示（IR）+ 可验证变换 + 可追溯持久化"的能力集合，用于把线性 token 推理与外部结构（树/图/表/索引/锚点）形成低摩擦闭环。
> 
> **类比**：辅助皮层之于 LLM ≈ 纸笔/棋盘/CAD 之于人类
> 
> **不是**：不是"更强的工具"，而是"更自然的肢体"

### 三层架构

```
┌─────────────────────────────────────────────┐
│  Layer 3: Domain Cortices（领域皮层）        │
│  - AST Engine（代码）                       │
│  - Graph Transformer（图结构）              │
│  - Spatial Canvas（空间布局）               │
└────────────────────┬────────────────────────┘
                     │ 领域特定 API
┌────────────────────▼────────────────────────┐
│  Layer 2: Memory Cortex（记忆皮层）          │
│  - 状态持久化                               │
│  - 树形记忆                                 │
│  - 工作记忆缓存                             │
└────────────────────┬────────────────────────┘
                     │ 结构化存储
┌────────────────────▼────────────────────────┐
│  Layer 1: Interface Cortex（接口皮层）       │
│  - Token ↔ Structure 双向映射              │
│  - 意图识别                                 │
│  - 反馈格式化                               │
└─────────────────────────────────────────────┘
```

### 核心隐喻

| 隐喻 | 来源 | 含义 |
|------|------|------|
| **"为 AI 发明数学"** | Claude | 符号系统是压缩的人类经验；当 LLM 调用 `ExtractMethod` 时，它继承的是人类数十年的重构经验 |
| **"认知 HUD"** | Gemini | 辅助皮层像钢铁侠的头盔界面——在 Token 流之上叠加结构化图层 |
| **"Agent 的 IDE 内核"** | GPT | 让"注意力焦点 → 可用操作 → 可验证结果 → 可回放历史"成为默认路径 |
| **"反向赛博格"** | 监护人 | 不是人类用 AI 增强自己，而是 AI 用外部工具增强自己 |

### 本体感 (Proprioception)

> **本体感**：Agent 知道自己"在哪里"、"做过什么"、"能引用什么"的能力。
> 
> **类比**：人类闭上眼睛也能知道自己的手在哪里；但 LLM 是"缸中之脑"，没有这种感知。

**Agent HUD 的四个组件**：

1. **状态栏 (Self-State)**：Token 余额、当前约束
2. **小地图 (Environment State)**：当前文件、Git 分支、最近修改
3. **任务日志 (Intent State)**：主任务、子任务、进度
4. **近期反馈 (Short-term Feedback)**：上一个操作的结果

## 设计原则

### P1. 双向映射

辅助皮层必须支持 Token 序列与结构化表示的双向映射：

```
Token 序列 ←→ 结构化表示（树/图/AST）
```

### P2. 最小认知负担

API 应该对 LLM "认知友好"：
- 操作语义清晰
- 参数简单
- 反馈明确

### P3. 渐进增强

辅助皮层是**增强**而非**替代**：
- LLM 仍可直接操作文本（降级模式）
- 辅助皮层提供更精确的结构化操作（增强模式）

### P4. 先闭环、再智能、再透明

GPT 的工程建议：
1. 先做可追溯的协议闭环
2. 再做智能化的推送和推断
3. 最后追求使用时的透明感
