## 🚀 一页纸版（3分钟上手）

### 30秒鸟瞰：畅谈会是什么？

**畅谈会是动态循环，不是瀑布流程**

```python
while True:
    snapshot = analyzeScene()           # 分析场势
    action = decideNextAction(snapshot) # 决策下一步
    nextSpeaker = selectSpeaker(action)
    
    if nextSpeaker.isAI():
        runSubagent(nextSpeaker)        # 调用 AI
        continue                        # ← 继续循环！
    
    if goalAchieved(): break
```

### 2分钟操作：时间轴三阶段

**开场前** → **进行中** → **收尾时**

#### 1. 开场前（准备）
- 创建会议文件：`agent-team/meeting/YYYY-MM-DD-<topic>.md`
- 填写背景、目标、参与者
- **开场问题**：用开放式问题邀请探索（如"这让你想到了什么？"）

#### 2. 进行中（动态循环）
- **场势快照**（每轮更新）：
  ```markdown
  #### 场势快照
  - **共识**：此刻我们已经同意什么？
  - **分歧**：最大的分歧是什么？（没有就写"无"）
  - **层级**：当前在哪个Tier？是否存在错位（如A谈Craft，B谈Resolve）？
  - **缺口**：还缺哪条关键输入/验证？需要谁补？
  - **能量**：讨论节奏是过热/平稳/疲惫？
  ```
- **选择动作**：基于场势决定下一步（邀请/追问/总结/派工/实施/收尾）
- **邀请发言**：使用`runSubagent`，指定`chatroomFile`和`appendHeading`

#### 3. 收尾时（收敛）
- **结论同步块**（必须）：
  ```markdown
  #### 结论同步块
  
  **✅ 已定**：
  - ...
  
  **❓ 未定**：
  - ...
  
  **📋 行动项**：
  - ...
  ```
- **状态标注**：在文件末尾标注`状态：已完成 ✅`

### 30秒自检：失稳信号 + 托住动作

| 失稳信号 | 托住动作 |
|:---------|:---------|
| **失焦**：连续10分钟绕圈无增量 | 场势快照 + 回到核心问题 |
| **迷失**：参与者不知在讨论什么 | 澄清问题 + 提供启动钩子 |
| **过热**：对立升级或连续打断 | 时间盒 + 冷却 + 结构性总结 |
| **过冷**：60秒无人接话 | 换视角邀请 + 认知温度计提示 |

## 📋 合规门槛（5条MUST）

> **构成性规则**：没有这些，畅谈会不成立。每条必须有可检验证据。

### MUST-1：唯一会议记录载体
- 畅谈会 **MUST** 有唯一的会议记录文件（chatroom file）
- 所有发言 **MUST** 以`### <Name> 发言`追加到该文件
- **证据**：文件存在；发言格式正确；无插入/重排既有发言

### MUST-2：动态循环推进
- 每次 subagent 返回后，主持人 **MUST** 重新分析场势并决定下一步
- **证据**：每轮至少出现一次"场势快照"或等价结构，并紧随"下一步动作"

### MUST-3：场势快照维护
- 畅谈会 **MUST** 显式维护场势快照结构（共识/分歧/缺口/能量）
- **证据**：快照至少出现1次；字段完整或明确标注N/A

### MUST-4：失稳托住执行
- 出现失稳信号时，主持人 **MUST** 执行托住动作并留下证据
- **证据**：`失稳信号:`与`托住动作:`成对出现

### MUST-5：结论同步块结束
- 畅谈会 **MUST** 以结论同步块结束（已定/未定/行动项）
- **证据**：文件末尾存在三段式结论同步块

## 🧭 详细指南（15分钟自如）

### 开场操作

**创建会议文件**：
```markdown
# 畅谈会：<主题>

> **日期**：YYYY-MM-DD
> **标签**：#review | #design | #decision | #jam
> **主持人**：<姓名>
> **参与者**：<名单>
> **状态**：进行中

---

## 背景
<核心问题>

## 目标
<期望达成的目标>

## 💬 畅谈记录
```

**开场问题设计**：
- 💡 **判断参照**：问自己"我的开场是在**邀请探索**还是**布置任务**？"
- 💡 **信号→问题→动作**：
  - Signal：参与者反应平淡或直接执行指令
  - Question："我是否过早引入了约束或视角？"
  - Action：改用开放式问题（如"这让你想到了什么？"）

**避免角色偏见**：
- 💡 **判断参照**：问自己"我是否在基于AGENTS.md的标签邀请特定视角？"
- 💡 **信号→问题→动作**：
  - Signal：总是邀请同一Agent做同一类事情
  - Question："我是否固化了Agent的角色认知？"
  - Action：使用Lens-Orchestra协议，让Agent自由选择透镜

### 进行中操作

**场势快照使用**：
- 💡 **判断参照**：当你感觉"刚才聊了很多但我记不清"时，是更新场势快照的好时机
- 💡 **默认值+覆盖**：默认每轮更新；当讨论高度聚焦时可跳过1轮

**选择下一步动作**：

| 动作 | 何时使用 | 具体做法 |
|:-----|:---------|:---------|
| **邀请** | 需要特定视角/专长 | 说明"为什么此刻邀请此人"+期望产出形式 |
| **探测** | 层级过热或错位 | 邀请相邻层级透镜进行"冷却"或"验证"（跨层探测） |
| **追问** | 观点有价值但不够具体 | 要求补证据/反例/边界条件 |
| **总结** | 散点观点需要收拢 | 命名共识+压缩分歧 |
| **派工** | 需要定制信息但不值得全员讨论 | 委派 Subagent 做摘要/对比/草拟 |
| **实施** | 方案已明确且需具体产出 | 邀请专家修改代码/文档 |
| **收尾** | 结论已足够 | 同步最终决议+确认行动项 |

**邀请发言格式**：
```yaml
# @Seeker
taskTag: "#review"
chatroomFile: "agent-team/meeting/xxx.md"
appendHeading: "### Seeker 发言"
scope: "术语一致性，不做实现建议"
outputForm: "Markdown要点列表"
```

### Lens-Orchestra 协议（无偏见邀请）

基于`lens-orchestra-protocol.md`，为避免角色偏见，推荐使用以下无偏见邀请方式：

**核心原则**：
- **锚点而非边界**：名字是起点，不是能力限制
- **选择而非指派**：Agent自由选择透镜，主持人只给锚点问题
- **可见而非纠正**：让视角选择和漂移可见，但不强制纠正
- **熔断与折返**：允许用低层透镜（如Craft）反证高层不可行，发起合法的"向上折返"

**无偏见邀请示例**：
```markdown
### 主持人邀请 (刘德智)

现在邀请@Curator：

**锚点问题**：
1. 从用户体验角度看，这个设计有什么优化空间？
2. 如何平衡功能性与易用性？
3. 对于新用户来说，最可能遇到的障碍是什么？

请遵守Lens-Orchestra协议：开头做透镜声明，结尾给接力请求。
请自由选择任何你认为相关的角度。
```

**避免的偏见邀请**：
```markdown
# ❌ 避免这样邀请
请@Curator从UX/DX角度分析这个设计...
```

**Lens-Orchestra协议格式**：
```markdown
[透镜：控制悖论]
...发言内容...

[换镜：认知模态]
...继续分析...

[接力：请下一位从规范核查角度审视这个建议]
```

**渐进式披露**：
- **Level 0**（最小闭环）：开头`[透镜：X]` + 结尾`[接力：Z]`
- **Level 1**（漂移可见）：可选中途`[换镜：Y]`
- **Level 2**（复盘友好）：会后结构化摘要

### 收尾操作

**结论同步块模板**：
```markdown
#### 结论同步块

**✅ 已定**：
- ...

**❓ 未定**：
- ...

**📋 行动项**（owner / 截止 / 验收）：
- ...

**🔺 待裁决**（需要人类/外部输入）：
- ...
```

**状态标注**：
- 进行中：`状态：进行中`
- 已完成：`状态：已完成 ✅`
- 已放弃：`状态：已放弃 ❌`（需说明原因）

### 特殊情况处理

**僵局（Stalled）**：
- 💡 **信号**：分歧超2轮，双方已说尽理由
- 💡 **动作**：升级决策（请监护人裁决）或重构问题（拆分/改验收/换视角）

**跑题（Derailed）**：
- 💡 **信号**：偏离核心问题超过5分钟
- 💡 **动作**：重述核心问题+显式丢弃次要支线

**能量管理**：
- 🔥 **过热**："现在讨论很热烈，我们可以继续发散"
- 🌡️ **平稳**："我们已经有很多想法了，是不是可以开始收敛？"
- ❄️ **过冷**："这个话题似乎有点冷，需要换个角度吗？"

### 透镜调色板（5色可选）

| 颜色 | Tier | 核心问题 | 体验锚点问句 |
|:-----|:-----|:---------|:-------------|
| 🔴 **价值红** | Resolve | 这为什么值得做？ | "如果我们不做这个，会错过什么？" |
| 🟡 **边界黄** | Shape | 用户看到什么？ | "用一句话，你会怎么向朋友介绍这个功能？" |
| 🟢 **约束绿** | Rule | 什么是合法的？ | "这个条款，你能想出一个违反它的例子吗？" |
| 🔵 **路径蓝** | Plan | 走哪条路？ | "如果只有两周时间，你会先做哪部分？" |
| 🟣 **实作紫** | Craft | 怎么造出来？ | "这段代码，半年后的你能看懂吗？" |

**使用方式**：
- 💡 **判断参照**：问自己"此刻引入透镜是在**提供选项**还是**限制视角**？"
- 💡 **信号→问题→动作**：，或出现层级错位
  - Question："我们是否需要换个认知层面看这个问题？"
  - Action：提供调色板选项，或进行跨层探测（"Plan层很完美，但请换上Shape透镜确认是否偏离初衷
  - Action：提供调色板选项（"你觉得此刻最关键的颜色是什么？"）

### 风格参照（情境-人物映射）

| 检查项 | Yes/No/NA | 证据（记录中的文本位置） |
|:-------|:----------|:-------------------------|
| MUST-1：唯一会议记录载体？ | | |
| MUST-2：动态循环推进？ | | |
| MUST-3：场势快照维护？ | | |

| MUST-5：结论同步块结束？ | | |

### 学习路径

**渐进式掌握**：
1. **一页纸版**（本文件前两节）→ 能主持一场畅谈会
2. **详细指南**（本文件第三节）→ 能自如应对各种情况
3. **工具箱**（本文件第四节）→ 能灵活使用增强工具
4. **设计哲学**（独立文档）→ 能理解为什么这样设计

**进阶信号**：
- "我遇到了一页纸没覆盖的情况" → 阅读详细指南
- "我想知道为什么要这样设计" → 阅读设计哲学
- "我想定制自己的主持风格" → 探索工具箱
