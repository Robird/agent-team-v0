## 核心隐喻

### 1. 动态循环 vs 瀑布流程

**瀑布流程的局限**：
- 预编排顺序：A→B→C→D
- 假设我们知道答案，只是需要执行
- 适合确定性任务，不适合探索性讨论

**动态循环的本质**：
```python
while True:
    snapshot = analyzeScene()           # 分析场势
    action = decideNextAction(snapshot) # 决策下一步
    nextSpeaker = selectSpeaker(action)
    
    if nextSpeaker.isAI():
        runSubagent(nextSpeaker)        # 调用 AI
        continue                        # ← 继续循环！
    
    if goalAchieved(): break
```

**关键区别**：每个 AI 发言后**重新分析场势、重新决策**，而不是按预定剧本推进。

### 2. 认知场域的园丁

**旧隐喻（交通警察）**：
- 控制流向：A该走了，轮到B
- 执行规则：红灯停，绿灯行
- 结果：秩序但僵化

**新隐喻（园丁）**：
- 提供条件：土壤、阳光、水分
- 观察生长：这棵苗需要什么？
- 适度干预：修剪过密的枝叶
- 结果：有机生长，自然涌现

**园丁的三项工作**：
1. **识别场势**：读懂"此刻需要什么"
2. **调节节奏**：太热时降温，太冷时加温
3. **保护空间**：确保每个视角都有机会被听见

### 3. 被托住的自由

**自由的两极**：
- 完全自由 → 可能迷失、冲突、无果
- 完全控制 → 丧失创造性、参与感

**被托住的自由**：
- **自由**：思考和表达不受限制
- **托住**：确保不会跌落（失焦、冷场、冲突）

**体验三要素**：
1. **安全感**：像在篝火旁聊天，而非站在讲台上汇报
2. **惊喜感**：像爵士乐的即兴——你抛出一个旋律，别人接上变奏
3. **成就感**：像登山后在山顶看到的风景——"原来从这里看是这样的"

## 设计原则

### 1. 开放性问题优先

**问题**：错误的引导视角不如开放性问题。

**原则**：
- 第一轮提问 **SHOULD** 使用开放性问题
- "请阅读前人的发言，再追加你的想法"
- "这让你想到了什么？"
- "从你的角度看，最关键的是什么？"

**理由**：开放式问题激活"探索"模式，任务指令激活"服从"模式。

### 2. 透镜是邀请，不是牢笼

**调色板隐喻**：
- ❌ 滤镜菜单：选一个，只能看到一种颜色
- ✅ 调色板：颜色都在那里，你可以随时取用，也可以混合

**使用方式**：
- "你觉得此刻最关键的颜色是什么？"
- "我们要不要换个角度看看？"
- 参与者可自选、混用、切换

### 3. 构成性规则最少化

**两类规则**：

| 类型 | 定义 | 处理方式 |
|:-----|:-----|:---------|
| **构成性规则** | 没有它，游戏不成立 | MUST，数量极少（≤5条） |
| **规约性规则** | 让游戏玩得更好 | 改写为"判断参照" |

**构成性规则判据**（满足≥2条）：
1. **身份判据**：违反则不能称为"畅谈会"
2. **闭环判据**：缺失会导致流程无法自洽
3. **可审计判据**：能被第三方仅凭产物进行二值验收

### 4. 判断参照优于硬规则

**问题**：过多的 MUST/SHOULD 可能抑制主持人的判断力。

**解决方案**：SQA 判断参照卡
- **Signal**（可观察信号）
- **Question**（主持自问）
- **Action**（可执行动作）

**示例**：
- ❌ "主持人 MUST 以开放式问题开场"
- ✅ **Signal**：参与者反应平淡或直接执行指令  
- ✅ **Question**："我是否过早引入了约束或视角？"
- ✅ **Action**：改用开放式问题（如"这让你想到了什么？"）

## 认知模型

### 1. 认知模态匹配

基于 Artifact-Tiers 框架：

| Tier | 认知模态 | 思维姿态 | 适合的提问方式 |
|:-----|:---------|:---------|:---------------|
| **Resolve** | Judicative → Commissive | 判断性→承诺性 | "这为什么值得做？" |
| **Shape** | Commissive → Descriptive | 承诺性→描述性 | "用户会看到什么？" |
| **Rule** | Descriptive → Normative | 描述性→规范性 | "这能写成测试吗？" |
| **Plan** | Normative → Strategic | 规范性→策略性 | "走哪条路？" |
| **Craft** | Strategic → Executable | 策略性→可执行性 | "具体怎么实现？" |

**关键洞见**：在 Resolve-Tier 用 Rule-Tier 的规范性思维处理判断性问题，就是认知模态错配。

### 2. 场势快照的四维模型

**场势快照不是状态报告，而是判断面板**：

| 维度 | 要回答的问题 | 认知功能 |
|:-----|:-------------|:---------|
| **共识** | 此刻我们已经同意什么？ | 建立共同基础 |
| **分歧** | 最大的分歧是什么？ | 识别冲突点 |
| **缺口** | 还缺哪条关键输入/验证？ | 发现盲区 |
| **能量** | 讨论节奏是过热/平稳/疲惫？ | 感知情绪温度 |

**设计理由**：把隐性情境显性化，让判断有依据。

### 3. 失稳托住机制

**失稳信号设计原则**：
1. **可观察**：能被第三方识别，不依赖读心
2. **可操作**：对应明确的托住动作
3. **防滥用**：避免主持人随意介入

**四类失稳信号**：
- **失焦**：连续10分钟绕圈无增量 → 结构性不强
- **迷失**：参与者不知在讨论什么 → 认知负荷超载
- **过热**：对立升级或连续打断 → 情绪失控
- **过冷**：60秒无人接话 → 参与度下降

**托住动作设计**：优先结构性动作（回顾、总结、时间盒、澄清问题），而非内容性裁决。

## 体验目标

### 1. 学习曲线设计

**渐进式掌握路径**：

```
一页纸版 → 详细指南 → 工具箱 → 设计哲学
   │           │          │          │
   └─ 3分钟    └─ 15分钟  └─ 按需    └─ 深度阅读
      能上手       能自如       能应变       能设计
```

**每个阶段的"毕业条件"**：
- **一页纸版**：完成一次畅谈会，产出结论同步块
- **详细指南**：能独立处理僵局、跑题、能量管理
- **工具箱**：能根据情境选择合适的透镜/风格
- **设计哲学**：能提出畅谈会规范的改进建议

### 2. 链接而非内联

**文档结构原则**：
- 每个层级的文档自包含当前层级的所有必要信息
- 通过链接指向下一层级，而非把下一层级的内容内联进来
- 读者按需深入，不被迫承载所有信息

**隐喻**：像游戏的难度选择——
- Easy（一页纸）：新手教程，手把手带你完成第一关
- Normal（详细指南）：完整规则，让你能通关
- Hard（工具箱）：高级技巧，让你玩得漂亮
- Expert（设计哲学）：Mod 工具，让你能修改游戏

### 3. 时间轴组织

**使用者心智模型**：主持人是按时间顺序思考的。

**文档组织**：
- ❌ 按抽象程度：规范层/工具层/体验层/证据层
- ✅ 按使用时机：开场前/进行中/收尾时 + 附录

**理由**：与自然认知顺序一致，即时相关性高，渐进式披露。

## 防僵化设计

### 1. Opt-in Lens（透镜可选原则）

**问题**：把透镜变成必经门会导致新的僵化。

**原则**：
- 透镜是"可选工具"，不是"必经门"
- 除非出现失稳信号，否则主持人 **MUST NOT** 强行上透镜
- 参与者可拒绝透镜，选择自由探索

### 2. Two-Phase Flow（两阶段流）

**流程设计**：
- **Phase A**：自由涌现（不贴标签、不归类）
- **Phase B**：必要时结构化（引入调色板/锚点/总结）

**可判定点**：记录中能看到从自由到结构化的显式转折。

### 3. Intervention Budget（介入预算）

**问题**：主持人过度介入会破坏"被托住的自由"。

**约束**：
- 主持人每 10 分钟最多 1 次"方向性介入"
- 其余介入优先用"结构性托住"

**目的**：把"园丁"从"频繁修剪"拉回"提供土壤"。

### 4. 防漂移检查点

**精简时的五个检查点**：

1. **三要素覆盖检查**：一页纸必须覆盖动态循环、场势快照、失稳托住
2. **闭环完整性检查**：是否仍形成"开场→循环→收敛"的最小闭环
3. **规范语气边界检查**：MUST ≤5条，隐喻不得承载 MUST
4. **SSOT 漂移检查**：核心定义只能有一个权威表述位置
5. **回归样例检查**：拿真实会议记录做回归测试
