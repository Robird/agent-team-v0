# F-001: HeadLen 下限阈值不一致

**状态**: ✅ 已解决（v0.15）
**位置**: rbf-format.md §6.1 `[R-REVERSE-SCAN-ALGORITHM]`
**类型**: Contradiction
**描述**: 逆向扫描算法中使用 `headLen < 16` 作为过滤条件，但 §3.3 声明最小 HeadLen = 20
**风险**: 实现者可能接受非法的帧长度（16-19），导致越界读取或数据污染
**建议**: 根据帧格式公式验证正确的最小值，并统一文档表述
