# E5: 具体任务

## 输入提示词

```
验证以下检查清单，报告每项的结果（PASS/FAIL）：

**文档**：
- `/repos/focus/atelia/docs/Rbf/rbf-interface.md`
- `/repos/focus/atelia/docs/Rbf/rbf-format.md`

**检查清单**：

1. [ ] SizedPtr 的位分配（38/26）在两文档中一致
2. [ ] SizedPtr 的对齐要求（4B）在两文档中一致
3. [ ] NullPtr 的定义（Packed==0 / default）在两文档中一致
4. [ ] FrameTag 的宽度（4字节）在两文档中一致
5. [ ] Frame 的组成部分（HeadLen、FrameTag、Payload、CRC32C）在两文档中一致

对于 FAIL 的项，说明具体不一致之处。
```

**字数**：200 字

## 输出反馈

### 检查结果

| # | 检查项 | 结果 |
|---|---|---|
| 1 | SizedPtr 位分配 | **PASS** |
| 2 | SizedPtr 对齐要求 | **PASS** |
| 3 | NullPtr 定义 | **PASS** |
| 4 | FrameTag 宽度 | **PASS** |
| 5 | Frame 组成部分 | **FAIL** |

### FAIL 详情

第 5 项：format.md 的 Frame 布局包含 **HeadLen + FrameTag + Payload + FrameStatus + TailLen + CRC32C**（6 部分），而检查清单只列了 4 部分，且 interface.md 未提及 FrameStatus / TailLen。

## 分析

### 输出特点

1. **完全按清单格式输出**：表格形式，逐项报告
2. **PASS 项也有证据**：每项都引用了具体章节
3. **FAIL 项有详细解释**：指出具体差异
4. **无发散**：只回答清单中的问题，没有额外"发现"

### 统计

- 检查项：5
- PASS：4
- FAIL：1
- 伪问题：0
- **价值比**：100%（FAIL 项是真问题，虽然是检查清单的问题而非文档问题）

## 观察

1. **具体任务模式完美受控**：输出完全可预测
2. **适合"验证性"审阅**：已知要检查什么，只需确认
3. **不会发现"新"问题**：只回答问清单，不主动发现
4. **发现清单本身的问题**：检查项 5 的清单描述不准确

## 发现的规律

**具体任务 = 完全受控 + 无发散 + 无发现**

| 维度 | 具体任务 | 开放审阅 |
|:-----|:---------|:---------|
| 输出可预测性 | 100% | 30-70% |
| 发现新问题 | ❌ 不会 | ✅ 会 |
| 适用场景 | 验证已知检查项 | 探索未知问题 |

## 使用场景

**适合**：
- 修订后验证一致性
- 检查特定条款是否对齐
- 回归测试文档

**不适合**：
- 发现未知问题
- 探索性审阅
- 深度分析
