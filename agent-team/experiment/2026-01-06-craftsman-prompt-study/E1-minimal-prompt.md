# E1: 极简提示词

## 输入提示词

```
审阅以下两个文档的一致性：

- `/repos/focus/atelia/docs/Rbf/rbf-interface.md`
- `/repos/focus/atelia/docs/Rbf/rbf-format.md`

列出发现的问题。
```

**字数**：47 字

## 输出反馈

### Sev2（3 个）

1. **docId 冲突** — 两个文档 docId 都是 `rbf-format`，interface 应该改为 `rbf-interface`
2. **Markdown 截断** — format.md 变更表 0.16 行链接未闭合
3. **章节编号不连续** — interface 缺 §6，format 缺 §9

### Sev3（3 个）

4. **"实现/依赖方向"歧义** — "format 实现 interface"表述有歧义
5. **标题/层级不自洽** — title 是 Shape-Tier 但 docId 是 format
6. **SizedPtr 定位混杂** — 接口语义 vs 线格式编码交叉

## 分析

### 有价值问题

| 问题 | 价值判断 | 理由 |
|:-----|:---------|:-----|
| docId 冲突 | ✅ 有价值 | 会导致 DocGraph 索引错误 |
| Markdown 截断 | ✅ 有价值 | 文档结构损坏，需修复 |
| 章节编号不连续 | ⚠️ 中等 | 影响引用，但不影响功能 |
| 实现/依赖方向 | ❌ 伪问题 | 表述问题，不影响理解 |
| 标题/层级不自洽 | ❌ 伪问题 | 元信息差异，不影响内容 |
| SizedPtr 定位混杂 | ⚠️ 中等 | 有道理但不紧急 |

### 统计

- 总问题数：6
- 有价值问题：2（docId、Markdown）
- 中等价值：2（章节编号、SizedPtr）
- 伪问题：2（表述、元信息）
- **价值比**：33%（严格）/ 67%（含中等）

## 观察

1. **默认行为倾向技术细节**：docId、Markdown 格式等是真问题
2. **仍有表述类问题**：但比例不高（33%）
3. **没有"证据链"类问题**：这次没有追问"证据的证据"
4. **输出结构清晰**：按严重性分级，有具体位置

## 初步假设

- 极简提示词让 Craftsman 关注"技术层面"而非"形式层面"
- 可能是因为没有明确要求"证据完整性"等抽象概念
- 后续可尝试进一步限定范围
