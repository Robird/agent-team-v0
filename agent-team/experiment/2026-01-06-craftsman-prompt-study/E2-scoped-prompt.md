# E2: 限定范围

## 输入提示词

```
审阅以下两个文档中 **SizedPtr 相关内容**的一致性：

- `/repos/focus/atelia/docs/Rbf/rbf-interface.md` 的 §2.3 SizedPtr 定义
- `/repos/focus/atelia/docs/Rbf/rbf-format.md` 的 §7 SizedPtr Wire Format

只关注 SizedPtr 的定义、位分配、语义在两文档之间是否一致。

列出发现的不一致问题。
```

**字数**：120 字

## 输出反馈

### 问题清单（3 个）

1. **粒度描述不一致** — interface 缺失"4B 粒度"说明，format 明确了
2. **Length 包含关系** — interface 精确定义"含 HeadLen 到 CRC32C"，format §7.1 可被误读
3. **NullPtr 用途承诺** — interface 扩展为"未找到/失败"，format 只定义"无效引用"

## 分析

### 有价值问题

| 问题 | 价值判断 | 理由 |
|:-----|:---------|:-----|
| 粒度描述不一致 | ✅ 有价值 | 会导致实现者解码方式分歧 |
| Length 包含关系 | ✅ 有价值 | 影响实现一致性 |
| NullPtr 用途承诺 | ⚠️ 中等 | 语义层面差异，不影响格式 |

### 统计

- 总问题数：3
- 有价值问题：2
- 中等价值：1
- 伪问题：0
- **价值比**：67%（严格）/ 100%（含中等）

## 观察

1. **限定范围有效减少问题总数**：6 → 3
2. **伪问题清零**：没有措辞、格式、元信息类问题
3. **问题质量显著提高**：都是实质性的技术不一致
4. **指定章节让 Craftsman 聚焦**：不再发散到 docId、章节编号等

## 发现的规律

**限定范围 = 减少发散 + 提高质量**

| 维度 | E1（极简） | E2（限定范围） |
|:-----|:-----------|:---------------|
| 总问题数 | 6 | 3 |

## 提示词技巧

1. **明确指定章节**：`§2.3 SizedPtr 定义` 比 `SizedPtr 相关内容` 更精确
2. **说明审阅维度**：`定义、位分配、语义` 限定了关注点
3. **排除干扰**：`只关注...` 避免发散
