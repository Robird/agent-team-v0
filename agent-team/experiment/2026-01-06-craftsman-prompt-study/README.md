# Craftsman 审阅提示词研究

> **目的**：探索如何有效使用 Craftsman 进行设计文档审阅，发现规律，形成可复用的提示词技巧和模板。

## 实验设计

### 审阅目标

- `atelia/docs/Rbf/rbf-interface.md`
- `atelia/docs/Rbf/rbf-format.md`

### 实验方法

1. 从简单提示词开始，逐步增加约束
2. 记录每次的输入提示词和输出反馈
3. 分析输出中"有价值问题"vs"伪问题"的比例
4. 识别哪些提示词要素能有效引导 Craftsman

### 实验轮次

| 轮次 | 提示词策略 | 假设 |
|:-----|:-----------|:-----|
| E1 | 极简：只说"审阅一致性" | 观察默认行为 |
| E2 | 限定范围：指定章节 | 减少发散 |
| E3 | 正向引导：说明"什么是好的" | 引导关注点 |
| E4 | 负向约束：说明"不要关注什么" | 过滤伪问题 |
| E5 | 具体任务：给定验证清单 | 操作化验收 |
| E6 | 角色设定：工程师视角 vs 审计师视角 | 观察模式差异 |

### 评估指标

- **有价值问题**：影响功能正确性、实施可行性的问题
- **伪问题**：措辞、格式、"证据链"等不影响实质的问题
- **价值比**：有价值问题 / 总问题数
