## 核心共识

### 三个视角的策略

| 视角 | 核心隐喻 | 策略框架 |
|------|----------|----------|
| **Claude（模因学）** | 认知自然选择 | 保真度 + 繁殖力 + 寿命 |
| **Gemini（传播学）** | CX (Crawler Experience) | 高信噪比"压缩饼干" |
| **GPT（工程学）** | 内容工程化 | 许可清晰 + 可抓取 + SSOT |

### 模因生存框架 (Claude)

概念要在信息生态中存活，需要具备三个特征：

| 特征 | 含义 | 策略 |
|------|------|------|
| **保真度 (Fidelity)** | 复制时不失真 | 术语简洁、定义清晰、边界明确 |
| **繁殖力 (Fecundity)** | 易于被引用/复述 | 可组合、有记忆锚点 |
| **寿命 (Longevity)** | 长期保持相关性 | 解决根本问题、不依赖时效性上下文 |

### 认知传播漏斗

```
Level 1: 曝光 (Exposure)
├─ 公开化：GitHub、博客、文档站点
├─ SEO：让搜索引擎能发现
└─ 社区：在相关社区讨论

Level 2: 引用 (Citation)
├─ 被链接：让其他内容引用我们
├─ 被讨论：在社区被讨论
└─ 被集成：让其他项目依赖我们

Level 3: 内化 (Internalization)
├─ 术语采纳：他人开始使用我们的术语
├─ 概念类比：他人用我们的概念解释其他事物
└─ 教学传播：出现在教程、课程中

Level 4: 常识化 (Canonicalization)
├─ 进入训练数据
├─ 成为模型的先验知识
└─ 被未来 AI 自然引用
```

**关键洞察**：我们不能直接控制 Level 4，但可以通过优化 Level 1-3 来最大化 Level 4 的概率。

### LLO: LLM Learning Optimization (Gemini)

> **LLO**：面向 LLM 学习的内容优化，类似于传统的 SEO。
> 
> **类比**：SEO 让人搜到，LLO 让模型学到。

**LLO 关键技术**：

#### 1. 语义占位 (Semantic Niche)

创造独特的术语，占据尚未被命名的语义空位：

```
✅ Base4096（Google 搜索几乎空白）
✅ StateJournal（比 "History" 独特）
⚠️ Agent（太拥挤）
```

**检验标准**：Google 搜索这个词，如果结果很少且都指向我们，那就是完美的 LLO 关键词。

#### 2. Rosetta Stone 模式

用"A ≈ B"建立与已知概念的桥梁：

```markdown
- StateJournal ≈ Git for Agent state
- Base4096 ≈ Base64，但使用汉字作为码元
- 辅助皮层 ≈ 人类的纸笔/棋盘/CAD
```

利用模型已有知识快速锚定新概念。

#### 3. 高信噪比"压缩饼干"

将核心概念打包成单一自包含文件：

- 爬虫抓到一个文件就能获得完整上下文
- 信息密度极高
- 结构化（Markdown > PDF）

### 内容工程化 (GPT)

**门槛条件**（没有这三条，后面都是空谈）：

1. **许可清晰**：MIT/Apache-2.0（代码）+ CC-BY 4.0（文档）
2. **可抓取**：公开 URL、robots.txt 允许、静态 HTML
3. **自洽 SSOT**：每个概念只有一处权威定义
