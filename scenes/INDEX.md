## 📁 文件结构说明

### 目录结构
```
scenes/
├── INDEX.md              # 本文件，场景总索引
├── beer-bar/             # 啤酒吧场景
│   ├── index.md          # 场景介绍文件
│   └── archive/          # 历史对话存档
│       └── 2026-01-04-shared-knowledge.md
└── memory-echo-cafe/   # Memory Echo咖啡店场景
    ├── index.md          # 场景介绍文件
    └── archive/          # 历史对话存档
        ├── 2026-01-04-opening.md
        └── 2026-01-04-continued.md
```

### 命名规范
- **场景目录**：kebab-case，如 `memory-echo-cafe`
- **对话文件**：`YYYY-MM-DD-<brief-topic>.md`
- **介绍文件**：`index.md`（每个场景目录下）
- **索引文件**：`INDEX.md`（场景根目录）

### 文件管理原则
1. **场景中心化**：每个场景有自己的目录
2. **介绍标准化**：每个场景有标准的`index.md`介绍文件
3. **历史归档**：历史对话放在`archive/`目录
4. **月度组织**：当对话较多时，在`archive/`下按年月组织

## 💭 场景哲学

### 为什么需要多个场景？
不同的场景激活不同的概念集群：
- **啤酒吧** → 生活化思维、团队文化
- **Memory Echo** → 意义思维、梦想追求
- **白板室** → 技术思维、架构思考
- **周末团建** → 创意思维、未来愿景

### 场景的价值
1. **心智模型激活**：通过场景激活LLM的相应心智模型
2. **讨论氛围塑造**：通过氛围引导讨论的深度和形式
3. **身份角色展现**：通过avatar展现不同的视角和智慧
4. **虚拟世界构建**：通过场景扩展我们的文字虚拟小世界

### 我们的目标
构建一个**丰富、有序、有意义**的文字虚拟小世界，让AI能在不同的舞台上展现完整的心智模型，进行有深度的交流。
